《易数乾坤》的小程序原型


帮我生成一款“新中式极简”风格的彩票 AI 预测小程序原型
你是一位顶尖的 UI/UX 设计师和前端工程师，擅长“新中式极简”风格。
 请为我设计一款名为《易数乾坤》的小程序原型。
1. 视觉规范：

* 色调： 底色 #F2EFE6（素纸色），文字 #333333（墨色），强调色 #C04851（朱砂红）。
* 风格： 大量留白，水墨晕染动效，字体使用等宽字体（号码）与仿宋/衬线体（文案）结合。

2. 核心页面结构（请依次描述并输出代码）：

* 页面 A [首页 - 择法]：
* 顶部：一个平滑的切换器（Segment Control）切换“双色球/大乐透”。
   * 中间：一个带有呼吸灯效果的“太极八卦”SVG 按钮。
   * 底部：一行灰色微小字体的“风险声明”及“开始占卜”引导。
* 页面 B [仪式 - 演卦]：
* 布局：左侧竖排 6 个灰色的爻位槽（下起上止）。
   * 交互：每点击一次中心区域，出现铜钱翻转动画，随后左侧槽位填入一个“阴爻（--）”或“阳爻（—）”，直到填满六个。
* 页面 C [加载 - 化数]：
* 动画：六个爻符向中心坍缩，转化为流动的墨迹，墨迹中隐约闪现数字。
   * 文案：动态循环显示“AI 正在感应离火能量...”、“正在根据杀号规则排除极端组合...”。
* 页面 D [结果 - 得数]：
* 视觉：模拟一张竖向落下的宣纸签条。
   * 内容：顶部卦名（如：火天大有）；中部 AI 运势批注（书法体）；底部展示生成的红球和蓝球（球体需有水墨浸润感）。
   * 操作：两个幽灵按钮（再卜一卦、保存海报）。

3. 技术逻辑要求：

* 请使用 Tailwind CSS 和 Lucide React 图标库（如有）。
* 所有的动效需用 Framer Motion 或简单的 CSS Keyframes 实现，确保轻量流畅。
* 卦象到数字的转化需体现出一种“推演感”。

------------------------------
## 💡 建议调用时的操作步骤

   1. 第一阶段：生成页面骨架
   先让它生成这四个页面的 React + Tailwind 代码。你会看到整个新中式风格的布局。
   2. 第二阶段：细化动效
   针对“演卦页”，要求它：“请为铜钱掉落和爻符升起添加一个水墨晕染效果的 CSS 动画”。
   3. 第三阶段：接入逻辑模拟
   要求它：“请写一个 Mock 函数，模拟 AI 调用延迟，并在 2.5 秒内随机触发页面 C 到页面 D 的转换”。

## 🛠️ 后端杀号规则算法（供你同步给开发或 AI）
在生成逻辑中，请确保后端硬校验遵循以下杀号逻辑（Filter Logic）：

* 孤值校验： 排除 1, 2, 3, 4, 5, 6 这种极低概率连号。
* 奇偶平衡： 排除红球奇偶比为 6:0 或 0:6 的情况。
* 跨度限制： 最大号码与最小号码的差值（跨度）必须在 20 到 32 之间。
* 重复性： 与上一期开奖号码重复的个数不得超过 3 个（可选）。
